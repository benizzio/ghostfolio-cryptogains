# Bug Report: Remove Obsolete hledger Boundary

**Type**: Spec conflict
**Severity**: Medium
**Status**: Reported
**Feature**: 006-empirical-financial-tests (`006-tasks-impl` checkout)
**Reported**: 2026-06-13

## Description

After BUG-002, rotki is the official external oracle for FIFO, LIFO, HIFO, and Average Cost aggregate fixture regeneration, with Scope-Local Hybrid handled through a separate composite oracle that uses rotki-backed arithmetic where applicable. hledger no longer has a defined purpose in the feature or project.

The current artifacts still preserve hledger as a retained historical or auxiliary boundary, including hledger vendoring, generated journal inputs, hledger command wrappers, hledger normalization code, hledger-specific tests, and documentation references. That conflicts with the post-BUG-002 oracle direction and leaves unnecessary dependency, fixture, and compliance work in scope.

## Root Cause Analysis

BUG-001 replaced the hledger-only oracle assumption because hledger-backed empirical runs skipped most supported comparisons. BUG-002 then tightened rotki regeneration so fixtures must derive from verified untracked pinned rotki source execution through the local adapter boundary.

Those patches changed the oracle model but left hledger in the feature as retained historical or auxiliary infrastructure. That created an active-artifact conflict: the feature now has an official rotki oracle path, but the plan and tasks still require creation, maintenance, verification, and review of hledger artifacts that do not contribute to acceptance.

The root cause is incomplete cleanup after the dependency replacement. The artifacts generalized hledger into "external oracle" language but did not remove the obsolete hledger implementation, fixture, and compliance surface once rotki became the selected external oracle.
