# Bug Report: Empirical Oracle Suite Skips Most Fixtures

**Type**: Dependency issue
**Severity**: High
**Feature**: `006-empirical-financial-tests`
**Reported**: 2026-06-10
**Evidence**: `specs/006-empirical-financial-tests/skipped-empirical-tests-report.md`, generated from `go test ./tests/empirical -count=1 -v`

## Description

The empirical calculation suite currently loads 13 golden fixture subtests, but only 2 fixtures execute project calculation and oracle comparison. The other 11 fixtures are skipped before calculation and comparison run.

Executed fixture groups:

- FIFO standard case.
- LIFO standard case.

Skipped fixture groups:

- Average Cost: 4 fixtures skipped because report output does not preserve case-slice pool provenance.
- HIFO: 2 fixtures skipped because persisted hledger oracle precision differs from calculation-layer financial normalization.
- Scope-Local Hybrid: 4 fixtures skipped because report output does not preserve comparable composition-rule provenance.
- Zero-priced FIFO: 1 fixture skipped because report output does not preserve comparable zero-priced lifecycle provenance.

This leaves most high-risk calculation methods and edge cases without empirical external-oracle comparison, despite the feature being marked through the empirical integration flow tasks. The report also identifies planned test fixes: remove zero-priced holding reductions from empirical oracle scope, replace hledger with a rotki-based external oracle for supported pure methods, use rotki ACB aggregate comparisons for Average Cost, replace HIFO fixtures with rotki HIFO fixtures, and validate Scope-Local Hybrid through a separate composite oracle.

## Root Cause Analysis

The original specification and plan assumed hledger could act as the external calculation oracle for all supported cost-basis methods and relevant edge cases. Implementation results show that assumption does not hold for the current empirical fixture set.

Observed causes by method group:

- Average Cost requires pool provenance that current report output and hledger fixture output do not preserve in a comparable case-slice form.
- HIFO fixture values generated from hledger differ from calculation-layer financial normalization enough that the suite skips the method instead of comparing it.
- Scope-Local Hybrid contains project-owned routing and lifecycle rules that no single hledger method models natively; current fixtures do not expose comparable composition-rule provenance.
- Zero-priced holding reductions are project-specific lifecycle behavior that the external oracle does not model faithfully enough for empirical comparison.

The current skip policy avoids false comparisons, but it also allows the empirical suite to pass without validating the methods and edge cases that the feature was created to cover. This is a dependency issue with implementation-drift consequences: the selected external dependency does not support the assumed validation contract, and completed tasks now overstate empirical coverage.
