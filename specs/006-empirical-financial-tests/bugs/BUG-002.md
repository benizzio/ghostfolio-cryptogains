# Bug Report: Rotki Fixture Regeneration Does Not Execute Pinned Source

**Type**: Spec conflict
**Severity**: High
**Status**: Patched
**Patched**: 2026-06-12
**Feature**: 006-empirical-financial-tests (`006-tasks-impl` checkout)
**Reported**: 2026-06-12

## Description

The rotki-backed oracle path was implemented as committed rotki empirical data or normalization inputs plus generated golden fixtures. That is insufficient. Golden fixture regeneration must execute a project-owned local oracle-generation boundary against pinned rotki source code, obtain oracle data from rotki calculation code, and then transform that oracle data into committed normalized golden fixtures.

To avoid vendoring rotki code in this repository, the regeneration command must download the pinned rotki source into an untracked project-local folder at generation time, verify the configured provenance and checksums, use that source through the local test-time adapter boundary, and keep only the resulting project-owned golden fixtures and provenance metadata under version control. Normal fixture-backed empirical test runs should continue to read committed golden fixtures without downloading rotki or invoking oracle generation.

## Root Cause Analysis

BUG-001 remediation allowed a bootstrap shortcut: committed raw rotki outputs or stable normalization inputs could satisfy the repository-controlled rotki boundary. That weakened the oracle requirement enough for implementation tasks to be marked complete without proving that golden fixtures can be regenerated from actual rotki calculation code.

The artifacts also conflate normal fixture-backed empirical tests with explicit fixture regeneration. A blanket prohibition on fetching external oracle source during oracle generation conflicts with the required non-vendored rotki model, where the regeneration command must fetch pinned source into an untracked local folder. The intended boundary is neither vendored rotki source nor developer-local rotki installation. It is a pinned, verified, untracked rotki source acquisition step used only by explicit golden-fixture regeneration.
