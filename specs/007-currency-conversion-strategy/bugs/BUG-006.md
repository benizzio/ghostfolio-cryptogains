# Bug Report: Converted Activities Render As Same Currency In Asset Details

**Type**: Implementation drift  
**Severity**: Medium  
**Feature**: 007-currency-conversion-strategy  
**Reported**: 2026-06-28  
**Status**: Reported

## Description

Generated report asset detail sections can label activities as `same currency` even when those same activities were actually converted. The contradiction is observable because the affected activity `Source ID` appears in `Currency Conversion Audit` with valid conversion evidence, correctly reflecting the original selected activity currency, report base currency, rate date, rate value, quote direction, original amount, and converted amount.

Expected behavior: an activity detail row whose `Source ID` appears in `Currency Conversion Audit` must be rendered as a converted activity, not as a same-currency activity. The `same currency` label must be reserved for activity values whose selected activity currency already matched the selected report base currency and therefore did not require exchange-rate conversion.

## Artifact Traceability

### spec.md

- **Affected user story**: User Story 1 - Select A Report Base Currency; User Story 2 - Use Official Historical Conversion Rates.
- **Affected requirements**: User Story 1 acceptance scenarios 3 and 4, User Story 2 acceptance scenario 6, FR-007, FR-008, FR-018, FR-019, FR-020, FR-021, FR-026, SC-001, SC-003, and SC-006.
- **Gap identified**: No primary spec gap. FR-021 already requires the report to distinguish same-currency activity values from converted activity values. The observed report output violates that requirement by rendering converted activity details as same-currency while the audit section proves conversion occurred.

### contracts/markdown-report.md

- **Affected sections**: `Monetary Tables`, `Conversion Audit Section`, and `Empty Audit Behavior`.
- **Impact**: The contract requires same-currency activity rows to remain distinguishable from converted rows and allows same-currency rows to be excluded from the conversion audit only if activity detail tables clearly mark them as same-currency. It does not currently include an explicit cross-section consistency assertion that any `Source ID` present in `Currency Conversion Audit` must not be labeled `same currency` in asset detail sections.

### data-model.md

- **Affected sections**: `SelectedActivityMonetaryContext`, `ConvertedActivityAmount`, `ConversionAuditEntry`, and `CapitalGainsReport`.
- **Impact**: The data model already differentiates `same_currency` from `converted` through conversion status and requires `ConversionAuditEntry` for converted priced activities. The rendering path appears to lose or misread that status when building asset detail sections, even though audit entries retain the correct conversion evidence.

### plan.md

- **Affected sections**: Summary, Constraints, Conversion Boundary And Rounding, Integration Anticorruption Layer, and Testing Strategy.
- **Impact**: The plan already states that same-currency rows do not request rates and converted rows resolve rate evidence before calculation. It needs a patched testing and rendering requirement that asset detail sections use the original conversion classification, not the post-conversion report-base currency value, when displaying same-currency versus converted status.

### tasks.md

- **Affected tasks**: T026, T027, T033, T042, T045, T066, T083, T084, T085, WU08, WU11, WU15, and WU16.
- **False completions**: T066 is marked complete but did not prevent converted activities from being labeled as same-currency in asset detail sections. T045, T083, T084, and T085 are marked complete but did not ensure consistency between grouped audit rendering and asset detail rendering. T026, T027, and T042 may need review if the implementation fix requires preserving conversion status from calculation through report artifacts.
- **Missing tasks**: Add regression tests that cross-check asset detail sections against `Currency Conversion Audit` by `Source ID`; add renderer unit coverage that rejects `same currency` labels for audited converted activities; add implementation tasks to preserve and render conversion status consistently from calculated artifacts into asset detail output.

## Root Cause Analysis

The specification and contract already define the required behavior, so this is implementation drift rather than a missing requirement. The report has two inconsistent rendering paths: `Currency Conversion Audit` uses conversion evidence that correctly identifies converted source activities, while asset detail sections render those same activities as `same currency`.

The likely implementation error is that asset detail rendering infers same-currency status from the rendered calculation currency or post-conversion amount currency. After conversion, converted monetary values are expressed in the report base currency, so that inference can incorrectly classify converted rows as same-currency. The detail renderer must instead use the original selected activity currency versus report base currency, or an explicit conversion status propagated from calculation artifacts.
