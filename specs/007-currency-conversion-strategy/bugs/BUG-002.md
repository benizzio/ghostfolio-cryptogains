# Bug Report: Federal Reserve Live DDP Integration Returns Malformed Rate

**Type**: Dependency issue  
**Severity**: High  
**Feature**: 007-currency-conversion-strategy  
**Reported**: 2026-06-21
**Status**: Patched
**Patched**: 2026-06-23

## Description

The opt-in external integration test for the Federal Reserve H.10 provider fails when running `make test-external-integration`. ECB live lookup passes, but the Federal Reserve EUR-to-USD live lookup fails with:

```text
lookup live provider evidence: conversion failed: reason=malformed_rate source_currency=EUR report_base_currency=USD activity_date=2024-01-06 provider=federal_reserve_h10
```

The live Federal Reserve host returns the Data Download Program HTML home page for the current client URL shape:

```text
https://www.federalreserve.gov/datadownload/?from=2023-12-07&to=2024-01-06&rel=H10&filetype=csv
```

The CSV mapper then attempts to parse HTML as Federal Reserve H.10 CSV and reports malformed rate evidence.

The Federal Reserve DDP direct download flow requires `Output.aspx` plus the H.10 daily-rates package identifier and layout parameters. A verified live CSV URL shape is:

```text
https://www.federalreserve.gov/datadownload/Output.aspx?rel=H10&series=60f32914ab61dfab590e0e470153e3ae&lastobs=&from=2023-12-07&to=2024-01-06&filetype=csv&label=include&layout=seriesrow&type=package
```

The current mapper fixtures also do not match the live DDP `seriesrow` CSV shape. The mapper expects rows shaped like `Currency,Monetary unit,Quote direction,<date>`, while the live DDP `seriesrow` package returns metadata columns such as `Descriptions:`, `Unit:`, `Multiplier:`, `Currency:`, `Unique Identifier:`, and `Series Name:` before date columns.

The committed live test expected value for EUR on `2024-01-05` is also stale for the current DDP source. The test expects `1.0946`; the current DDP daily-rates package returns `1.0957`. The specification and research require regenerated reports to use currently published authorized rates when the TUI-session cache does not already contain defensible same-key evidence.

## Root Cause Analysis

The Federal Reserve implementation was validated against deterministic fixture CSV that does not match the live Data Download Program direct-download contract. The client uses the DDP landing path with query parameters instead of the documented automated direct-download URL shape. The live service responds with HTML, and that response is then classified as malformed provider evidence.

The external test also retained a Federal Reserve EUR value that differs from the currently published DDP daily-rates package. The active spec and research explicitly prefer current DDP observations over unrevised historical release archive pages, so the expected value must follow the current DDP source used by the production client.

This bug exists because deterministic unit and contract fixtures proved quote-direction and conversion behavior, but they did not mirror the actual Federal Reserve DDP endpoint and CSV envelope closely enough. The opt-in live external integration suite correctly exposed the gap.
