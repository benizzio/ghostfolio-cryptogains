# Bug Report: Currency Conversion Audit Table Includes Redundant Provider Columns

**Type**: Spec conflict  
**Severity**: Medium  
**Feature**: 007-currency-conversion-strategy  
**Reported**: 2026-06-23  
**Status**: Patched
**Patched**: 2026-06-24

## Description

The current `Currency Conversion Audit` table is too large and repeats provider-level information in every conversion amount row. Topic 2 requests removing these columns from the audit table:

- `Rate Authority`
- `Rate Kind`

The audit table should use this column order instead:

```markdown
| Date | Source ID | Asset | Amount Kind | Rate Date | Source Currency | Original Amount | Report Base Currency | Converted Amount | Quote Direction | Rate Value |
```

The requested output keeps the conversion-specific evidence needed for each row, including amount kind, original amount, converted amount, rate date, quote direction, and rate value. Provider-level metadata such as authority and rate kind should be disclosed once in `Rate Source Summary`, not repeated as per-amount audit columns.

## Root Cause Analysis

The active specification and Markdown contract carried provider metadata into every conversion audit row. That conflicts with the provider-level `Rate Source Summary` boundary introduced for the related source-summary bugfix. The table also appears to follow the accumulated evidence model shape instead of the user-facing audit workflow, which makes the row harder to scan.

This is a spec-artifact conflict rather than implementation drift because the current artifacts explicitly require the oversized table. The implementation can be corrected only after the spec, contract, and tasks define the compact audit table contract.
