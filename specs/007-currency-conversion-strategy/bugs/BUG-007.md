# Bug Report: Asset Detail Currency Columns Are Ambiguous

**Type**: Spec gap  
**Severity**: Medium  
**Feature**: 007-currency-conversion-strategy  
**Reported**: 2026-06-28  
**Status**: Patched
**Patched**: 2026-06-28

## Description

Generated report `Asset Detail` sections are unclear because activity currency information is not named or placed precisely enough for mixed-currency reports.

Required behavior:

- In the `In-Year Activity` table, rename `Activity Currency` to `Original Activity Currency`.
- In the `In-Year Activity` table, render columns in this exact order: `Date`, `Source ID`, `Type`, `Quantity`, `Unit Price`, `Gross Value`, `Fee`, `Quantity After Row`, `Basis After Row`, `Original Activity Currency`, `Calculation Currency`, `Conversion Status`, and `Note`.
- In the `Liquidation Calculations` table, remove the `Activity Currency` column because the original activity currency is already disclosed in the preceding activity table and does not add useful liquidation-calculation evidence.

## Artifact Traceability

### spec.md

- **Affected user story**: User Story 1 - Select A Report Base Currency; User Story 2 - Use Official Historical Conversion Rates.
- **Affected requirements**: User Story 1 acceptance scenarios 3, 4, and 5; User Story 2 acceptance scenario 6; FR-004, FR-018, FR-019, FR-020, FR-021, FR-026, SC-001, and SC-003.
- **Gap identified**: The spec requires report currency disclosure, preservation of original selected currency, and distinction between same-currency and converted rows, but it does not define the `Asset Detail` table contracts that make those disclosures clear. It does not require the `In-Year Activity` column name `Original Activity Currency`, does not define the exact `In-Year Activity` column order, and does not prevent duplicated or low-value `Activity Currency` content in `Liquidation Calculations`.

### contracts/markdown-report.md

- **Affected sections**: `Monetary Tables`.
- **Impact**: The contract says activity currency columns continue to show the selected activity currency before conversion and calculation currency columns show the selected report base currency. It does not define the precise Asset Detail table headers or column order, and it does not state that liquidation calculation rows must omit a repeated `Activity Currency` column.

### data-model.md

- **Affected sections**: `SelectedActivityMonetaryContext`, `ConvertedActivityAmount`, `ConversionAuditEntry`, and `CapitalGainsReport`.
- **Impact**: The model carries enough information to render original selected currency, calculation currency, and conversion status. The gap is in the report rendering contract, not in the stored model fields. The model documentation may still need wording alignment so `original_currency` maps to report-visible `Original Activity Currency` in activity detail rows.

### plan.md

- **Affected sections**: Summary, Project Structure, Conversion Boundary And Rounding, Integration Anticorruption Layer, and Testing Strategy.
- **Impact**: The plan covers Markdown rendering for report currency, provider summaries, conversion audit, and asset detail conversion labels. It needs an explicit rendering-contract update for `Asset Detail` table headers, ordering, and liquidation-column removal.

### tasks.md

- **Affected tasks**: T033, T045, T066, T078, T079, T083, T093, T094, T096, WU11, and WU16.
- **False completions**: T045 and T096 are marked complete but do not cover the newly required `Asset Detail` column names and ordering. T066, T083, T093, and T094 are marked complete but only guard same-currency versus converted consistency, not the clarity and bloat issues described here. T033, T078, and T079 are marked complete but focus on `Currency Conversion Audit`, not `Asset Detail` table contracts.
- **Missing tasks**: Add Markdown contract tests for exact `In-Year Activity` header order and `Original Activity Currency` naming; add renderer unit tests for the same header order; add contract and renderer tests proving `Liquidation Calculations` omits `Activity Currency`; add renderer implementation work to update the two Asset Detail tables.

## Root Cause Analysis

The feature has evolved from conversion correctness into user-facing audit clarity. Existing bugfixes already addressed conversion audit row grouping and converted-versus-same-currency labeling, but the spec artifacts still describe Asset Detail currency disclosure only at a general level. Because the exact table contract is absent, implementation can satisfy the broad currency-disclosure requirements while still producing confusing column names, suboptimal column order, and repeated currency content in liquidation calculations.
