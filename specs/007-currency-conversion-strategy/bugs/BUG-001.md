# Bug Report: Coverage Artifacts Written To Project Root

**Type**: Spec conflict  
**Severity**: Medium  
**Feature**: 007-currency-conversion-strategy  
**Reported**: 2026-06-21

## Description

Final coverage verification for the active feature is producing `coverage.cov` and `coverage.xml` in the repository root. The correct placement is the already existing `dist/coverage` folder, which the repository uses for generated coverage artifacts as `dist/coverage/coverage.out` and `dist/coverage/coverage.xml`.

The active feature artifacts currently direct final coverage commands to root-level paths. This creates new generated files in the project root and diverges from the maintained repository coverage-output contract.

## Artifact Traceability

### spec.md

- **Affected user story**: Cross-cutting validation for User Story 1, User Story 2, and User Story 3.
- **Affected requirements**: Testing impact requirement, FR-030, Financial Calculation Evidence, and SC-001 through SC-007 validation evidence.
- **Gap identified**: The specification requires project-owned test and coverage evidence, but it does not state that generated coverage artifacts must use the maintained `dist/coverage` output location. This omission allows feature-local verification artifacts to conflict with the repository's established generated-output location.

### plan.md

- **Affected sections**: Technical Context, Testing Strategy, and final coverage-gate validation references.
- **Impact**: `plan.md` currently records coverage enforcement with `go test ./... -covermode=atomic -coverprofile=coverage.cov` and `gocoverageplus` repository gates. That root-level profile path conflicts with the maintained repository coverage artifact paths used by `make coverage` and `tools/coveragegate`.

### tasks.md

- **Affected tasks**: T059, T060, and WU23.
- **False completions**: T059 and T060 are marked complete, but the task wording validates creation of root-level `coverage.cov` and `coverage.xml` rather than the maintained `dist/coverage/coverage.out` and `dist/coverage/coverage.xml` artifacts.
- **Missing tasks**: Add remediation tasks to update final coverage commands to write under `dist/coverage`, remove root-level generated coverage artifacts, regenerate coverage in `dist/coverage`, and verify the repository coverage gate against the maintained paths.

## Root Cause Analysis

The active feature artifacts copied or retained older root-level coverage command examples instead of using the repository's current generated-output convention. The repository already documents and enforces `dist/coverage/coverage.out` and `dist/coverage/coverage.xml`, while the active feature's plan, quickstart, and tasks reference `coverage.cov` and `coverage.xml` in the project root.

This is a spec-artifact conflict rather than an implementation-only bug because the implementation followed commands from the active feature artifacts. The artifacts need to be corrected before final verification is repeated.

## Recommended Fix

1. Run `/speckit.bugfix.patch` to update the active feature artifacts so coverage output consistently uses `dist/coverage/coverage.out` and `dist/coverage/coverage.xml`.
2. Run `/speckit.bugfix.verify` to confirm the spec, plan, quickstart, and tasks are consistent.
3. Resume `/speckit.implement` to remove root-level generated coverage files and rerun coverage through `make coverage` or equivalent commands that target `dist/coverage`.
