# Bug Report: PDF Layout Has Width, Spacing, And Section Formatting Defects

**Type**: Implementation drift
**Severity**: High
**Feature**: 008-report-pdf-annex
**Reported**: 2026-07-09
**Status**: Patched
**Patched**: 2026-07-09

## Description

After the BUG-003 PDF renderer changes, the generated PDF is no longer a plain line dump, but the visible layout still has multiple production readability defects.

In `Ghostfolio Capital Gains And Losses Report`:

- Most tables are too wide for the portrait page, are compressed even with smaller fonts, do not respect right padding, and are cut on the right. The PDF should use landscape orientation for all pages.
- The `Gains-And-Losses Summary` subtitle and the preceding `Report Calculation Currency: <currency code>` line overlap because the vertical margin between text sections is negative or too small.
- `Overall Yearly Net Total` should be the last line of the `Gains-And-Losses Summary Table`, but it is rendered outside the table.
- `Rate Source Summary` incorrectly renders as a `Rate Source Summary Table`; it should render bold classifier label lines followed by non-bold values, matching the properly rendered Markdown report.
- `Reference Section` includes an unnecessary `Reference Table` subheading.
- `Asset Detail: <asset symbol>` subheadings have too little or negative top margin and touch or invade previous sections.
- `In-Year Activity` subheadings on the same page have negative or insufficient top margin and overlap the previous section.

In `Annex 1 - Audit`:

- `Asset: <asset symbol>` subheadings on the same page have too little top margin and appear too close to the previous section's table.

## Root Cause Analysis

BUG-003 moved the PDF renderer toward structured `gopdf` layout primitives, but the implementation and tests still permitted a layout that is technically structured while visually invalid for the report's table width and section flow. The renderer appears to use portrait page dimensions or insufficient available width for the report tables, inadequate vertical spacing before section headings, and generic table/subheading helpers in places where the Markdown report contract requires label/value lines or no extra subheading.

The primary root cause is implementation drift from the PDF readability contract, with a secondary specification gap around page orientation and exact spacing/fit expectations. Existing validation did not catch concrete visual defects after the renderer stopped being a line dump.
