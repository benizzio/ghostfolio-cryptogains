# Bug Report: PDF Uses Line Dumps Instead Of Human-Legible Layout

**Type**: Implementation drift
**Severity**: High
**Feature**: 008-report-pdf-annex
**Reported**: 2026-07-07
**Status**: Patched
**Patched**: 2026-07-07

## Description

After BUG-002, the generated PDF no longer exposes Markdown source syntax as report content. However, the PDF remains illegible for human review because the report data is dumped as simple line-oriented text without the visual hierarchy, sections, styled labels, and table layout expected from the properly rendered Markdown report.

Expected behavior is a human-legible A4 PDF whose titles, segments, section hierarchy, and tables follow the same presentation intent as the rendered Markdown report without using Markdown language or Markdown source syntax. Wide tables may continue on additional pages when needed, including horizontal page additions if required to keep rows and columns readable.

The non-negotiable technical requirement is that the implementation must use `github.com/signintech/gopdf` to render tables, headings, styled text, A4 pages, custom fonts, and table rows/columns. The PDF renderer must not satisfy the PDF path with plain line dumping, Markdown conversion, or a renderer that bypasses the planned `gopdf` layout boundary.

## Root Cause Analysis

BUG-002 corrected the renderer boundary enough to remove Markdown syntax from the PDF body, but the implementation and tests still allowed a minimal text-emission solution. That solution appears to serialize report-domain data as plain lines rather than rendering a human-legible document with headings, styled text, tables, rows, columns, and page-aware continuation.

The root cause is implementation drift from the intended PDF renderer contract, combined with insufficient test and manual validation coverage for readable PDF layout quality. The existing artifacts also need a targeted hardening patch because the new non-negotiable requirement names `github.com/signintech/gopdf` and specific layout primitives that must be used, not merely any PDF-specific text path.
