# Bug Report: PDF Section Spacing And Continuation Labels Regress

**Type**: Implementation drift
**Severity**: High
**Feature**: 008-report-pdf-annex
**Reported**: 2026-07-10
**Status**: Reported

## Description

The PDF renderer still has production readability defects after BUG-005:

- The `Gains-And-Losses Summary`, `Rate Source Summary`, `Reference Section`, `Asset Detail: <asset symbol>`, and `In-Year Activity` subheadings remain too close to their preceding segments. Their top margin must be doubled from the current value.
- A table split across pages adds an unnecessarily verbose continuation label such as `Continued: Reference Section (continued)`. The correct label is `Reference Section (continued)`; the `Continued: ` prefix must not be rendered.
- A continuation label such as `Continued: Continued` is rendered when no table has split across pages. Continuation context must appear only on a genuine table continuation page.

## Root Cause Analysis

BUG-005 introduced a numeric minimum of 12 points and row-preflight behavior, but completed renderer tests and implementation did not validate the requested stronger visual spacing threshold. The renderer defines the affected spacing at 12 points, so it can pass the existing numeric test while still producing cramped output.

The continuation path also conflates page heading flow with table continuation. The renderer passes a generic `Continued` label to its continuation-page helper and prefixes continuation labels with `Continued: `. Existing tests encode that behavior, so the false completion is both implementation and regression-coverage drift. The prior artifacts require continuation context for continued content but leave the exact label and no-split suppression insufficiently specified.
